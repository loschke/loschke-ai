# Inhaltstiefe-Skalierung: Kombinierter Ansatz

Diese Demo zeigt die erweiterte Inhaltstiefe-Skalierung mit zwei verschiedenen Ansätzen:

1. **Abschnittsbasierte Skalierung**: Ganze Abschnitte werden ein- oder ausgeblendet
2. **Abschnittsinterne Skalierung**: Innerhalb eines Abschnitts werden Details skaliert

## Abschnittsinterne Skalierung

<SectionContent title="Was ist Inhaltstiefe-Skalierung?">
  <BaseContent>
    Die Inhaltstiefe-Skalierung ermöglicht es Lesern, die Detailtiefe eines Artikels selbst zu bestimmen.
  </BaseContent>
  
  <StandardContent>
    Mit einem einfachen Steuerungselement können Leser zwischen drei Stufen wählen: Key Insights, Standard und Detailliert.
    
    Diese Funktion ist besonders nützlich für lange Artikel, die sowohl für Leser mit wenig Zeit als auch für Leser mit tiefergehendem Interesse zugänglich sein sollen.
  </StandardContent>
  
  <DetailedContent>
    Die technische Umsetzung basiert auf MDX-Komponenten, die es ermöglichen, Inhalte mit verschiedenen Tiefenstufen zu kennzeichnen. Die Steuerung erfolgt clientseitig mit JavaScript, wobei die Benutzereinstellung im localStorage gespeichert wird.
    
    Für Suchmaschinen und Screenreader bleibt der gesamte Inhalt zugänglich, da die Filterung nur visuell erfolgt.
  </DetailedContent>
</SectionContent>

<SectionContent title="Vorteile der Inhaltstiefe-Skalierung">
  <BaseContent>
    - Personalisierte Leseerfahrung
    - Bessere Zugänglichkeit für verschiedene Lesertypen
  </BaseContent>
  
  <StandardContent>
    - Reduzierte kognitive Belastung
    - Effizientere Informationsaufnahme
    - Höhere Leserabschlussrate
    
    Studien zeigen, dass Leser, die die Informationsdichte selbst steuern können, eine höhere Zufriedenheit und bessere Informationsaufnahme aufweisen.
  </StandardContent>
  
  <DetailedContent>
    - Verbesserte SEO durch längere Verweildauer
    - Geringere Absprungrate durch angepasste Inhaltstiefe
    - Möglichkeit, sowohl Einsteiger als auch Experten mit einem Artikel anzusprechen
    
    Eine A/B-Test-Studie von Nielsen Norman Group ergab, dass Websites mit skalierbaren Inhalten eine um 28% höhere Conversion-Rate aufweisen als solche mit statischen Inhalten.
  </DetailedContent>
</SectionContent>

<DepthContent depth={2}>
  ## Implementierungsdetails
  
  Die Implementierung der Inhaltstiefe-Skalierung erfordert folgende Komponenten:
  
  1. `ContentDepthControl.astro`: Steuerungselement für die Tiefenauswahl
  2. `DepthContent.astro`: Wrapper für abschnittsbasierte Skalierung
  3. `SectionContent.astro`: Container für abschnittsinterne Skalierung
  4. `BaseContent.astro`, `StandardContent.astro`, `DetailedContent.astro`: Komponenten für verschiedene Tiefenstufen innerhalb eines Abschnitts
  
  Die Steuerung erfolgt über JavaScript, das die Sichtbarkeit der Elemente basierend auf der gewählten Tiefenstufe regelt.
</DepthContent>

<DepthContent depth={3}>
  ## Technische Details und Erweiterungsmöglichkeiten
  
  ### JavaScript-Implementierung
  
  ```javascript
  function setDepth(depth) {
    const depthNum = parseInt(depth);
    
    // Abschnittsbasierte Elemente
    document.querySelectorAll('[data-content-depth]').forEach(element => {
      const elementDepth = parseInt(element.getAttribute('data-content-depth'));
      element.style.display = elementDepth <= depthNum ? 'block' : 'none';
    });
    
    // Abschnittsinterne Elemente
    document.querySelectorAll('[data-section-depth]').forEach(element => {
      const elementDepth = parseInt(element.getAttribute('data-section-depth'));
      element.style.display = elementDepth <= depthNum ? 'block' : 'none';
    });
  }
  ```
  
  ### Zukünftige Erweiterungen
  
  - Integration eines Schiebreglers für feinere Granularität
  - Automatische Generierung von Zusammenfassungen mit KI
  - Personalisierte Standardeinstellungen basierend auf Benutzerverhalten
  - Export-Funktion für die gewählte Tiefenstufe als PDF
</DepthContent>

## Kombinierter Ansatz in der Praxis

<SectionContent title="Anwendungsbeispiele">
  <BaseContent>
    Die Inhaltstiefe-Skalierung eignet sich besonders für:
    - Tutorials und How-To-Guides
    - Wissenschaftliche Artikel
    - Produktdokumentationen
  </BaseContent>
  
  <StandardContent>
    **Tutorials und How-To-Guides**
    
    Bei Tutorials können Anfänger die Grundschritte sehen, während fortgeschrittene Nutzer zusätzliche Details und Hintergrundinformationen erhalten.
    
    **Wissenschaftliche Artikel**
    
    Wissenschaftliche Artikel können sowohl eine Zusammenfassung für fachfremde Leser als auch detaillierte Methodik und Ergebnisse für Experten bieten.
  </StandardContent>
  
  <DetailedContent>
    **Produktdokumentationen**
    
    Bei Produktdokumentationen können verschiedene Zielgruppen bedient werden:
    
    - Level 1: Grundlegende Bedienung für Endnutzer
    - Level 2: Erweiterte Funktionen für Power-User
    - Level 3: Technische Details für Entwickler und Integratoren
    
    Ein Beispiel ist die Dokumentation von Microsoft Azure, die verschiedene Detailebenen für unterschiedliche Nutzergruppen anbietet.
  </DetailedContent>
</SectionContent>

## Fazit

<SectionContent>
  <BaseContent>
    Die Inhaltstiefe-Skalierung bietet eine flexible Möglichkeit, Inhalte für verschiedene Lesertypen zugänglich zu machen.
  </BaseContent>
  
  <StandardContent>
    Durch die Kombination von abschnittsbasierter und abschnittsinterner Skalierung können Autoren die optimale Struktur für ihre Inhalte wählen und Lesern eine maßgeschneiderte Erfahrung bieten.
    
    Die Implementation mit MDX-Komponenten ist relativ einfach und kann in bestehende Websites integriert werden.
  </StandardContent>
  
  <DetailedContent>
    Für Content-Ersteller bedeutet dies zwar einen höheren initialen Aufwand, da Inhalte für verschiedene Tiefenstufen aufbereitet werden müssen, jedoch führt dies langfristig zu einer höheren Reichweite und Nutzerzufriedenheit.
    
    Die Technologie kann weiter ausgebaut werden, beispielsweise durch KI-gestützte automatische Zusammenfassungen oder personalisierte Empfehlungen basierend auf dem Leseverhalten.
  </DetailedContent>
</SectionContent>
